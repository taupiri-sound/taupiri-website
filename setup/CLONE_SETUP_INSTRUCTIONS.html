1Clone Specific Branch and Create New GitHub Repository

IMPORTANT: This will clone only a specific branch without git history, then push it to a completely new GitHub repository.

Step 1a: Clone only the specific branch you want (without history):

# Replace 'branch-name' with the branch you want to clone
# Replace 'new-project-name' with your desired project folder name
git clone --depth 1 --branch <branch-name> <current-repo-url> <new-project-name>
cd <new-project-name>
Example: If you want to clone the 'development' branch:
git clone --depth 1 --branch development https://github.com/user/repo.git my-new-project

Step 1b: Remove existing git history and start fresh:

# Remove the existing git repository
rm -rf .git

# Initialize a new git repository
git init

# Add all files to the new repository
git add .

# Create your first commit with fresh history
git commit -m "Initial commit from template"

Step 1c: Create the new repository on GitHub (in your organization):

Option A: Using GitHub CLI (recommended):

# Install GitHub CLI if you haven't already: https://cli.github.com/

# Create a new PRIVATE repository under your organization
gh repo create <organization-name/new-repo-name> --private --source=. --remote=origin --push

# Or create a PUBLIC repository under your organization
gh repo create <organization-name/new-repo-name> --public --source=. --remote=origin --push
Example for organization:
gh repo create my-org/my-new-cms --private --source=. --remote=origin --push

Example for personal account:
gh repo create myusername/my-new-cms --private --source=. --remote=origin --push

Option B: Manual GitHub creation (organization):

  1. Go to your organization page: https://github.com/orgs/<organization-name>/repositories
  2. Click the green "New repository" button
  3. Enter your repository name
  4. Choose visibility (Private or Public)
  5. DO NOT initialize with README, .gitignore, or license
  6. Click "Create repository"
  7. Copy the repository URL (e.g., https://github.com/organization-name/new-repo.git)
# Connect to your new repository
git remote add origin <new-repo-url>

# Push to your new repository (creates main branch)
git push -u origin main
Note: You must be an owner or have appropriate permissions in the organization to create repositories.

Step 1d: Create the branch from the cloned code (optional but recommended):

# Create and push the branch that matches the one you cloned
# Replace 'branch-name' with the same branch you cloned earlier
git checkout -b <branch-name>
git push -u origin <branch-name>

# Switch back to main if desired
git checkout main
Example workflow:
If you cloned the 'development' branch:
git checkout -b development
git push -u origin development

Now your new repo has both main and development branches with the same code, but fresh git history.
What this approach gives you:
  • ‚úÖ Only the specific branch content you want
  • ‚úÖ Fresh git history starting from scratch
  • ‚úÖ New GitHub repository with your desired name
  • ‚úÖ Main branch with your code
  • ‚úÖ Optional matching branch name for consistency
  • ‚úÖ No old commits or history from the original repo

2Update Dependencies to Latest Versions

IMPORTANT: Install latest versions of all dependencies instead of using locked versions.

Delete existing dependency lock files:

rm package-lock.json

Update package.json with latest dependency versions:

# This command updates all dependencies to their latest versions
npx npm-check-updates -u

Install the updated dependencies:

npm install

Test that everything works:

npm run build
npm run typecheck

If there are any breaking changes from version updates, address them:

  • Check the build output for errors
  • Review dependency changelogs for breaking changes
  • Update code as needed for compatibility

  // Production domain - update this for your new project

üõ† Troubleshooting

Common Issues:

  • Sanity Studio not loading: Check your project ID and API tokens in .env.local
  • Build errors after dependency updates: Review breaking changes in updated packages
  • Domain not showing in robots.txt: Verify NEXT_PUBLIC_BASE_URL is set correctly
  • TypeScript errors: Run npm run typegen to regenerate Sanity types

Getting Help:

  • Check Sanity project settings at sanity.io/manage
  • Verify environment variables are correctly set
  • Ensure your Sanity project dataset exists and is accessible
